# Documentation Index

Use this index as the starting point for backend docs. Read the most specific doc first, then follow links outward when you need broader context.

## Canonical Precedence

**All documentation in `backend/docs/` is canonical.** Legacy or root-level documentation is archived and non-canonical. When docs conflict with implementation (routes, controllers, services, models), prefer the implementation and update the docs to match.

For governance policies on ownership, review cadence, and canonical precedence, see `backend/docs/governance/source-of-truth-map.md`.

## How to use docs

- Start with the topic area that matches your task.
- Prefer the canonical doc for the concept instead of duplicate summaries.
- Use the navigation links below to move from overview to detailed reference.
